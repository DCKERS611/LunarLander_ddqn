# 7.7-7.12 强化学习课程设计任务表

## 项目基本信息

| 项目 | 内容 |
|---|---|
| 课程设计方向 | 强化学习 |
| 项目题目 | 基于 Dueling Double DQN 与优先经验回放的 LunarLander 智能体训练研究 |
| 实验任务 | 训练智能体控制月球着陆器安全降落 |
| 实验环境 | Windows + Python + PyTorch + Gymnasium |
| 主要算法 | DQN、Double DQN、Dueling DQN、Prioritized Experience Replay |
| 最终目标 | 完成代码、训练、推理、实验分析、截图和课程报告 |
| 提交节点 | 7.12 前完成最终提交 |

## 7.7 任务：环境搭建与基础 DQN

### 当日目标

跑通强化学习环境，实现最小可运行的基础 DQN，并生成第一版训练曲线。

### 具体任务

| 顺序 | 任务 | 说明 | 状态 |
|---:|---|---|---|
| 1 | 创建项目目录 | 建立 `rl-lunarlander-ddqn` 或当前工程目录 | 待完成 |
| 2 | 创建 Python 虚拟环境 | 使用 `venv` 管理依赖 | 待完成 |
| 3 | 安装基础依赖 | PyTorch、Gymnasium、Pygame、Box2D、Matplotlib 等 | 待完成 |
| 4 | 测试 LunarLander 环境 | 确认 `LunarLander-v3` 能正常启动 | 待完成 |
| 5 | 准备保底环境 | 如果 Box2D 失败，切换到 `CartPole-v1` | 待完成 |
| 6 | 实现 Q 网络 | 输入状态，输出每个动作的 Q 值 | 待完成 |
| 7 | 实现经验回放池 | 保存 `(state, action, reward, next_state, done)` | 待完成 |
| 8 | 实现 epsilon-greedy | 平衡探索与利用 | 待完成 |
| 9 | 实现 target network | 固定目标网络，提升训练稳定性 | 待完成 |
| 10 | 跑基础 DQN | 训练 100-300 个 episode，确认 reward 正常记录 | 待完成 |

### 当日产出

```text

2. 可运行的 LunarLander 或 CartPole 环境
3. 基础 DQN 代码
```
